## 🧭 What are we building here?

During this journey, you will slowly build a real full-stack system.

You will start from the basics and move step by step into backend, frontend, databases, DevOps, testing, deployment, and a final demo.

# 🗺️ Roadmap

| Stage | Name | Main Focus | Output | DONE? |
|---|---|---|---|---|
| 0 | Setup | Terminal, VS Code, Git, npm, project structure | Ready dev environment + first repo | V |
| 1 | JavaScript Basics | Variables, loops, functions, arrays, objects, async/await | JS file managing users/tasks in memory | X |
| 2 | TypeScript | Types, interfaces, enums, typing, basic generics | Same task logic, but in TypeScript | X |
| 3 | First Backend API | Node.js, Express/Fastify, REST, routes, Postman | Basic tasks API in memory | X |
| 4 | Code Layers | Controller, service, repository, DTO, basic SOLID | Cleaner API structure | X |
| 5 | Errors & Logs | Validation, middleware, custom errors, logger, requestId | API with clear errors and logs | X |
| 6 | React Basics | Components, props, state, forms, events | Local task list screen | X |
| 7 | React + API | fetch/axios, React Query, loading/error states | UI connected to the API | X |
| 8 | PostgreSQL | SQL, tables, relations, migrations, seed | API connected to real DB | X |
| 9 | MongoDB Activity Log | Collections, flexible schema, activity events | Activity log endpoint | X |
| 10 | Redis Cache | Key-value, TTL, cache aside, invalidation | Cache for users/tasks | X |
| 11 | Elasticsearch | Indexing, mapping, search relevance | Search endpoint for tasks | X |
| 12 | Testing | Unit, integration, E2E, mocks, coverage | Test suite running in one command | X |
| 13 | Docker Compose | Dockerfile, containers, networks, volumes | Full system with `docker compose up` | X |
| 14 | Kubernetes / OpenShift | Pods, deployments, services, routes, secrets | App running in a cluster | X |
| 15 | Helm | Charts, templates, values, upgrade, rollback | Helm chart for the system | X |
| 16 | GitLab CI/CD | Pipelines, jobs, Docker build, deploy | Automated build/test/deploy | X |
| 17 | Kafka | Events, producer, consumer, retry, DLQ | User/task event flow | X |
| 18 | Storage | S3 vs NFS, upload, metadata, signed URLs | File upload demo | X |
| 19 | Observability | Logs, metrics, traces, requestId, dashboards | Basic production visibility | X |
| 20 | Final Capstone | Demo, docs, architecture, ownership | Final presentation + live demo | X |
